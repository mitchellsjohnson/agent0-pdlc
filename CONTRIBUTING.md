# Contributing to Agent0 PDLC

Thank you for your interest in contributing to Agent0 PDLC! This document provides guidelines for contributions.

---

## How to Contribute

### Reporting Issues

- Use GitHub Issues for bug reports and feature requests
- Search existing issues before creating a new one
- Provide clear reproduction steps for bugs
- Include your environment details (OS, tools, versions)

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feat/your-feature`
3. **Make your changes**
4. **Test your changes** thoroughly
5. **Commit with clear messages**: Follow conventional commits
6. **Push and create a Pull Request**

### Commit Message Format

```
<type>: <description>

[optional body]

[optional footer]
```

Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation only
- `style`: Formatting, no code change
- `refactor`: Code restructuring
- `test`: Adding tests
- `chore`: Maintenance

---

## Areas for Contribution

### High Priority

- **Additional Agent Templates**: AgentDocs, AgentOps, AgentDBA, etc.
- **Integration Guides**: More AI tools beyond Cursor and Claude
- **Language-Specific Templates**: Testing strategies for Python, Go, Rust, etc.
- **CI/CD Examples**: GitHub Actions, GitLab CI, Jenkins templates

### Documentation

- Tutorials and getting-started guides
- Video walkthroughs
- Translations
- FAQ improvements

### Tools & Integrations

- MCP server integrations
- IDE extensions
- Workflow automation scripts

---

## Code Style

### Markdown

- Use GitHub Flavored Markdown
- 4-space indentation for nested lists
- Use straight quotes, not smart quotes
- End files with a newline

### Code Examples

- Include working examples when possible
- Test all code snippets
- Use consistent formatting

---

## Review Process

1. All PRs require at least one review
2. CI must pass before merge
3. Documentation updates may be required
4. Breaking changes need discussion first

---

## Community Guidelines

- Be respectful and constructive
- Help others learn
- Credit others' work
- Follow the code of conduct

---

## Questions?

- Open a GitHub Discussion for questions
- Tag issues appropriately
- Be patient - maintainers are volunteers

Thank you for contributing!
